# Plan: 对比远端和本地，然后提交

## Goal

对比远端与本地差异，将工作区变更提交并推送到 origin/master。

## Current Understanding

- **远端 HEAD = 本地 HEAD**：都是 `892ba66`（Fix critical import paths and multiple bugs (3.0.3-alpha)），无需 pull
- 工作区有 9 个源文件变更（action.py / config.py / context_lookup.py / daily_memory.py / event_handler.py / privacy.py / stream_index.py / tools.py / wander.py）
- `__pycache__/` 已被 git 跟踪但 `.gitignore` 有 `__pycache__/` 规则，需要 `git rm --cached` 清理
- `.agents/context/` 下有新增 session 文件和 GOAL 文档，属于项目上下文，应一并提交
- 远端仓库：`https://github.com/fuilyha56-wq/send_to.git`

## Scope

### In Scope

- 从 git 索引中移除 `__pycache__/` 目录（保留本地文件）
- 暂存 9 个源文件变更
- 暂存 `.agents/context/` 下的新增/变更文件
- 提交并推送到 origin/master

### Out of Scope

- 修改任何源代码
- 修改 `.gitignore`
- 修改 manifest.json 版本号
- 运行测试

## Implementation Steps

### Step 1: 从 git 跟踪中移除 __pycache__

Expected behavior:
- `__pycache__/` 目录从 git 索引中移除，本地文件保留
- 后续 `__pycache__` 变更不再被 git 跟踪

Files / entry points:
- `__pycache__/` 目录（递归）

Implementation notes:
- 使用 `git rm -r --cached __pycache__/`

Acceptance criteria:
- `git status` 中 `__pycache__` 文件不再出现在 "Changes not staged" 中
- 本地 `__pycache__/` 文件仍然存在

### Step 2: 暂存所有变更

Expected behavior:
- 9 个源文件 + `.agents/context/` 文件 + `__pycache__` 删除记录全部暂存

Files / entry points:
- `action.py`, `config.py`, `context_lookup.py`, `daily_memory.py`, `event_handler.py`, `privacy.py`, `stream_index.py`, `tools.py`, `wander.py`
- `.agents/context/` 下所有文件
- `__pycache__/` 删除记录

Implementation notes:
- `git add action.py config.py context_lookup.py daily_memory.py event_handler.py privacy.py stream_index.py tools.py wander.py`
- `git add .agents/context/`

Acceptance criteria:
- `git status` 显示所有变更已暂存

### Step 3: 提交

Expected behavior:
- 创建一条有意义的提交信息

Implementation notes:
- 提交信息格式：`fix: 多项改进与配置优化`
- 包含变更摘要

Acceptance criteria:
- `git log -1` 显示新提交

### Step 4: 推送到远端

Expected behavior:
- 本地提交推送到 origin/master

Implementation notes:
- `git push origin master`

Acceptance criteria:
- 推送成功，远端与本地同步

## Edge Cases

- 如果推送被拒绝（远端有新提交），先 `git pull --rebase` 再推送
- 如果 `__pycache__` 已在 `.gitignore` 中但之前被跟踪，`git rm --cached` 后 `.gitignore` 规则生效

## Verification

Run:
- `git status` — 工作区干净
- `git log --oneline -3` — 确认新提交
- `git diff origin/master` — 确认与远端一致

## Stop Conditions

Stop and report if:
- `git push` 失败且 rebase 有冲突
- 任何 git 操作报错

## Reporting Requirements

Report:
- 修改的文件列表
- 提交 hash 和 message
- 推送结果
